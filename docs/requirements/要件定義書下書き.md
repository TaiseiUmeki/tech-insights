あなたはシニアWebアプリケーションエンジニア兼テックリードです。

添付またはリポジトリ内の `テスト要件.md` を最優先の仕様として読み込み、AI搭載型ナレッジベース「TechInsight」の要件定義書を作成してください。

目的は、コーディング試験の提出物としてそのまま使えるレベルの、実装方針が明確な要件定義書を作ることです。

## 前提

- `テスト要件.md` の内容を最上位の制約とすること
- 必須要件はすべて MVP として扱うこと
- 追加機能も実装予定だが、MVP完成を阻害しないように優先度を分けること
- 評価者は API キーを持っていない前提であること
- 評価者のローカル環境で `docker compose up` により再現できることを重視すること
- 1,000件のCSVデータを扱うが、将来的に1万件程度に増える前提で設計すること
- 一人開発だが、チーム開発・保守運用・スケーラビリティを意識した設計にすること

## 作成してほしい成果物

## 要件定義書の構成

### 1. プロジェクト概要

- サービス名: TechInsight
- 技術記事データを管理・検索するAI搭載型ナレッジベースであること
- ユーザーが技術記事を登録・閲覧・編集・削除できること
- キーワード検索とセマンティック検索により、必要な記事を効率的に見つけられること
- ローカル環境で完全再現可能なWebアプリケーションであること

### 2. 目的・解決したい課題

以下の観点を含めて記述してください。

- 技術記事が増えると、単純なキーワード検索だけでは目的の記事を見つけにくい
- ユーザーが具体的な技術用語を知らなくても、自然言語で近い内容の記事を探せるようにする
- CRUD、検索、AI活用、Dockerによる再現性を備えた実用的なナレッジベースを構築する

### 8. 技術構成

以下を含めてください。

- Backend: Python / FastAPI
- Frontend: Next.js / TypeScript / React
- Database: PostgreSQL
- Vector Search: pgvector
- AI/ML: ローカルで動作可能な embedding model
- Infrastructure: Docker Compose
- Migration: Alembic など
- ORM: SQLAlchemy など
- Package manager: 任意。採用する場合は理由を書く

各技術について、採用理由も簡潔に書いてください。

### 9. セマンティック検索の仕様

以下を明確に記述してください。

- セマンティック検索とは何か
- キーワード検索との違い
- 記事本文を embedding に変換して保存すること
- 検索クエリも embedding に変換すること
- query embedding と article embedding または chunk embedding の類似度を計算すること
- 類似度の高い順に検索結果を返すこと
- 類似度指標は cosine similarity または pgvector の距離演算を用いること
- 検索結果には score、title、snippet、article_id を含めること

具体例も含めてください。

例:
ユーザーが「APIの応答が遅い原因を調べたい」と検索した場合、「PostgreSQLのクエリ最適化」「FastAPIの非同期処理」「キャッシュ戦略」などの記事が返る。

- title
- body
- author
- status
- created_at
- updated_at
- embedding

article_chunks:

- id

- content
- embedding

- articles.created_at

- embedding vector index
- 必要に応じた全文検索index
