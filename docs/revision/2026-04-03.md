## 変更内容

### バックエンド

- ドメイン例外と一元的なエラーハンドリングを導入
  - Application 層の `HTTPException` をドメイン固有の例外に置換（`AuthenticationError`, `AuthorizationError`, `ResourceNotFoundError`, `ResourceConflictError`, `BusinessValidationError`, `BusinessLogicError`）
  - `presentation/handlers.py` で全例外を HTTP レスポンスに変換（401/403/404/409/422/400）
- ユーザー CRUD API を追加（認証必須の7エンドポイント）
  - `POST /users` — ユーザー作成（login_id 重複チェック付き）
  - `GET /users` — ユーザー一覧取得（ページネーション対応）
  - `GET /users/me` — ログイン中ユーザー情報取得
  - `GET /users/{user_id}` — ユーザー詳細取得
  - `PUT /users/{user_id}` — ユーザー更新
  - `DELETE /users/{user_id}` — ユーザー削除
  - `PUT /users/me/password` — パスワード変更
- DTO 変換パターンの統一（`from_dto()` / `from_entity()` クラスメソッド）
- テストアーキテクチャの改善
  - connection レベルのトランザクション rollback でテスト分離
  - 全レイヤー（domain/application/infrastructure/presentation）にテスト追加

### フロントエンド

- `entities/user` を追加（型定義 + API クライアント + テスト）
- `features/auth/get-current-user` を追加（React Query による `/users/me` 取得）
- `features/auth/auth-initializer` を追加（認証初期化ロジックを App 層から抽出）
- `widgets/sidebar` を追加（shadcn/ui Sidebar + ユーザーメニュー + ログアウト）
- `features/unsaved-change` を追加（未保存変更の検出・警告ダイアログ）
- UI ショーケースページを追加（`/dev/ui`）
- 共通ユーティリティを追加（`api-error-handler`, `nuqs-parsers`, `use-delayed-loading` 等）

### インフラ (CDK)

- Client VPN Stack を追加（オプション: `config.clientVpn.enabled`）
  - 相互証明書認証による VPN 接続、ALB への閉域アクセス
- Route 53 Stack を追加（オプション: `config.route53.enabled`）
  - 既存ホストゾーンへの ALB / CloudFront エイリアス A レコード
- ECS フロントエンド配信を追加（`frontend.type: 'ecs'`）
  - BackendStack の同一 ALB 上でパスベースルーティング（`/api/*` → バックエンド、`/` → フロントエンド）
- `config.vpc` を削除し `config.network` に統一（重複解消）
- アーキテクチャ図（draw.io）とリソース一覧（cdk-resources.md）を新規作成
- infra/README.md、infrastructure.md をリンク修正・全面更新
- Security Monitoring Stack を追加（オプション: `config.securityMonitoring.enabled`）
  - CloudTrail によるログ改ざん検知・不正アクセスのリアルタイム Slack 通知基盤
  - 監視項目: ログ改ざん操作（StopLogging, DeleteTrail 等）、Root ログイン、MFA 未使用ログイン、アクセス拒否スパイク
  - 通知フロー: CloudTrail → EventBridge/CloudWatch Alarm → SNS → Lambda → Slack
  - Construct 3 件（CloudTrail Trail / S3 バケット / CloudWatch MetricFilter+Alarm）、Resource 1 件、Stack 1 件、Lambda 1 件（slack-notifier）
  - テスト 3 件追加（Construct 2 件 + Stack 1 件）
- セキュリティ改善
  - Client VPN: 認可ルールを VPC 全体 → ALB パブリックサブネットのみに制限（最小権限）
  - CloudTrail バケット: 暗号化を SSE-S3 → SSE-KMS（AWS 管理キー）に変更
- Lambda を JS → TypeScript に変換（slack-notifier, api-resource）
  - NodejsFunction（esbuild）によるバンドル自動化

### CI/CD

- `showcase-sync-check.yml` を追加（UI コンポーネントのショーケース登録チェック）

### Makefile

- `make lint` / `make format` を Backend のみに変更
- `make lint-all` / `make format-all`（Backend + Frontend）を追加
- `make test-all`（Backend + Frontend テスト）を追加
- `make clean` / `make docker-clean` を追加

### ドキュメント
